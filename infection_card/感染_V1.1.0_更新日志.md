# 感染 V1.1.0 更新日志

本次更新重做了开场身份选择流程，并补齐了远程状态栏、身份选择器和背包分类。玩家进入新聊天后会直接看到身份选择器；选择预设身份会切换到对应开场，并自动写入身份、地点、技能和三栏背包。状态栏也更新为新的赛博终端风格，远程加载默认可用。

## 新功能与重要改进

### 开场与身份选择

- 重做开场第一页；现在新聊天第一条消息只显示身份选择器，玩家可以直接选择预设身份或填写自定义身份。
- 六个预设身份改为独立开场页；点击对应身份的 **「开始」** 后，会切换到该身份的完整开场，不再通过世界书关键词生成。
- 预设身份开场会自动初始化变量；选择身份后，状态栏可以读取到身份、开场地点、技能和背包。
- 自定义身份保留现场生成方式；填写身份、地点、物品、技能、经历和外观后，点击 **「开始」**，系统会发送一条完整开局信息。

### 状态栏与远程加载

- 状态栏更新为新的赛博终端视觉；背包、状态、人物、势力和主线分页仍按原入口查看。
- 远程状态栏和远程身份选择器默认启用；正常导入完整卡后，不需要手动修改远程地址。
- 远程地址已改为正式托管地址；现在状态栏和身份选择器会加载 GitHub Pages 上的正式页面，不再指向占位地址。

### 显示与变量体验

- 新增开场变量初始化脚本；预设身份开场后会自动读取本轮变量更新内容并写入变量系统。
- 新增变量更新块隐藏规则；预设身份开场中的变量初始化内容不会直接显示在正文里。
- 状态栏仍由 `<StatusPlaceHolderImpl/>` 加载，剧情开始后的回复需要继续在末尾输出状态栏占位符。
- 背包变量继续分为 **「武器弹药」**、**「普通物品」** 和 **「重要/关键物品」** 三栏；模型后续更新背包时，也会按这三类写入。

## 问题修复

- 修复身份选择器跳转错位的问题；现在 **「野外生存博主」**、**「海岛灯塔看守」** 等按钮会切换到正确身份开场。
- 修复开场第一页显示作者说明的问题；现在玩家不会先看到发布说明和导入提示。
- 修复自定义身份点击 **「开始」** 后多生成一层空白用户消息的问题；现在只会发送一条包含表单内容的用户消息。
- 修复预设身份开场后变量没有立即初始化的问题；现在身份、地点、技能和背包会进入变量系统。
- 修复预设身份初始物品全部进入 **「普通物品」** 的问题；现在旧式步枪、防火斧、多功能刀、巡护砍刀、捕兽夹和手持信号枪会进入 **「武器弹药」**，巡查图、影像存储卡、医疗记录、样本箱、航海图和密封箱会进入 **「重要/关键物品」**。
- 修复变量规则里背包路径不明确的问题；现在后续剧情获得弹药、武器、地图、记录或任务证据时，会按三栏背包分类更新，不会默认全部写进普通物品。
- 修复远程状态栏和身份选择器未默认启用的问题；现在重新导入完整卡后，远程页面会直接使用正式地址加载。

## 使用方法

1. 重新导入 `dist/infection-card-v1.1.json`。
2. 新开聊天，第一条消息会显示身份选择器。
3. 选择一个预设身份并点击 **「开始」**，或切到 **「自定义身份」** 填写表单后点击 **「开始」**。
4. 开场生成后，检查状态栏中的身份、地点、技能和背包是否已更新。
5. 如果你按部件导入，请同步更新状态栏页面、身份选择器页面、正则和开场变量初始化脚本。

## 兼容性说明

- 旧聊天：建议新开聊天使用 V1.1.0 的开场选择器和变量初始化流程。
- 重新导入：需要重新导入完整角色卡 JSON；如果按部件分发，也需要重新导入状态栏、身份选择器、开场变量初始化脚本和显示规则。
- 数据处理：本次没有删除旧存档变量字段；背包仍使用原来的三栏结构，只修正初始物品和后续更新规则的分类方式。
- 旧开局：已经开局的聊天不会自动重新分类旧消息里的初始物品；如果想使用新的初始背包分类，建议新开聊天重新选择身份。
